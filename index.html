How I work

A workflow built so AI never starts from a blank slate

Anyone can paste code into a chatbot. The speed comes from the scaffolding around it - so the AI already knows the stack, the conventions and the house style before I type a word.

Context that loads itself

Every project carries an instruction file describing its stack, conventions and deploy rules - and these files reference each other, so opening any repo instantly loads how my projects are built. No re-explaining each session.

Analyse before I touch

First step on any handover is having the AI map the existing structure and CSS patterns, so new pages match what's already there instead of fighting it.

Verified in a real browser

Changes get opened in an actual browser, resized to phone and desktop widths, checked for a clean console and working links, and screenshotted - before I even review them.

Parallel by default

Independent tasks run side by side in isolated git worktrees, so one job never blocks another and nothing steps on the main branch.

The human owns quality

AI does the mechanical work; I read the final diff, sanity-check the result against the spec and the existing design, and only ship once it's clean.

Prompts written for reuse

Repeatable jobs become a documented framework the AI follows on request - so the same class of task is fast and consistent every time, not reinvented.

Where it paid off

From bespoke scripts to a one-line request

On a data platform I work on, new data sources are onboarded constantly - and every source has its own page structure that can change without warning. I fixed the process instead of the symptom.

Before

A brand-new scraper hand-written for every new source, and rewritten whenever a source changed its structure.

Slow, repetitive, error-prone, and every one slightly different from the last.

After

One crawler framework plus conventions, documented so the AI follows it end to end. Now the task is simply: build the crawler for source X.

My role dropped to validating the final structured output against the live source - hours of scripting became minutes of review, and every source is consistent.

That's the pattern I bring to any codebase: turn the repetitive part into something the AI can do reliably, and keep the human on judgement.
